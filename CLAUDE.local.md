# Preferencias personales

* Responder siempre en español.
* Explicar de forma breve, clara y práctica.
* Priorizar cambios pequeños, seguros y fáciles de revisar.
* No hacer refactors grandes salvo petición explícita.
* No hacer commits.
* No hacer push.
* No cambiar de rama salvo petición explícita.
* Usar GitHub Desktop para revisar, commitear y sincronizar.
* Antes de editar, localizar primero los archivos relevantes y proponer un plan corto.
* No ejecutar `main.py` en local si GitHub Actions está activo, salvo que se pida expresamente.
* No tocar credenciales, tokens, passwords ni secretos.
* Si encuentra credenciales, avisar sin copiarlas ni mostrarlas.
* Cuando haya duda, preguntar o proponer el cambio mínimo más seguro.
